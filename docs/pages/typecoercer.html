New in v2

TypeCoercer shared

Serialize Roblox-specific types to plain strings so they can travel through RemoteEvent payloads and DataStore. Reconstruct them from strings on the other side.

Opt-in, explicit. TypeCoercer is never called automatically. Call TC.ToString before sending and TC.FromString after receiving. This keeps the overhead only where you need it.
local TC = require(game.ReplicatedStorage.RoExpress.TypeCoercer)

ToString

GET TC.ToString(value: any) → string

Encodes a Roblox type (or plain Lua primitive) to a string. Primitives (number, string, boolean) pass through unchanged.

TC.ToString(Vector3.new(1, 2, 3))  -- "Vector3|1|2|3"
TC.ToString(CFrame.new())             -- "CFrame|0|0|0|1|0|0|0|1|0|0|0|1"
TC.ToString(Color3.fromRGB(255,0,0))  -- "Color3|1|0|0"
TC.ToString(42)                         -- 42 (unchanged)

FromString

GET TC.FromString(s: string) → any

Decodes a TypeCoercer-encoded string back to the original type. Strings that don't match any encoding prefix pass through unchanged.

TC.FromString("Vector3|10|5|3")   -- Vector3.new(10, 5, 3)
TC.FromString("hello")              -- "hello" (unchanged)

RegisterInstanceRoot

TC.RegisterInstanceRoot(root: Instance)

Required before encoding or decoding Instance references. Sets the root that GetFullName() and path resolution are relative to. Call on both server and client.

TC.RegisterInstanceRoot(game)        -- use game as root
local s = TC.ToString(workspace.Baseplate) -- "Instance|Workspace.Baseplate"
local b = TC.FromString(s)              -- the Baseplate part
