New in v2

TypeCoercer shared

Serialize Roblox-specific types to plain strings so they can travel through RemoteEvent payloads and DataStore. Reconstruct them from strings on the other side.

Used automatically by the Router. When you declare a typed route param (:id=number, :mode=Enum.CameraType), the Router calls TypeCoercer internally to coerce and serialize values. You only need to call it directly when building URLs manually or working with DataStore.
local TC = RoExpress.TypeCoercer
-- or: require(game.ReplicatedStorage.RoExpress.TypeCoercer)

ToString

TC.ToString(value: any) → string

Serializes a value to its wire string. The Router calls this when building redirect URLs via RouteBuilder.build().

TC.ToString(Vector3.new(10, 5, -20))         -- "10,5,-20"
TC.ToString(Color3.fromRGB(255, 0, 128))     -- "255,0,128"
TC.ToString(CFrame.new(1,2,3))              -- "1,2,3,0,0,0"
TC.ToString(Enum.CameraType.Scriptable)      -- "Enum.CameraType.Scriptable"
TC.ToString(workspace.Baseplate)            -- "Part:Baseplate"
TC.ToString(42)                              -- "42"

FromString

TC.FromString(raw: string, typeName: string) → (boolean, any)

Reconstructs a value from its wire string. Returns (true, value) on success, (false, nil) on failure. The Router calls this automatically for typed params — you only need it when coercing manually.

TC.FromString("10,5,-20", "vector3")                    -- true, Vector3.new(10, 5, -20)
TC.FromString("255,0,128", "color3")                    -- true, Color3.fromRGB(255, 0, 128)
TC.FromString("Scriptable", "Enum.CameraType")           -- true, Enum.CameraType.Scriptable
TC.FromString("Enum.CameraType.Scriptable", "Enum.CameraType") -- true, same
TC.FromString("Enum.CameraType.Scriptable", "Enum")        -- true, any EnumItem
TC.FromString("Part:Baseplate", "Instance")             -- true, workspace.Baseplate
TC.FromString("abc", "number")                          -- false, nil

RegisterInstanceRoot

TC.RegisterInstanceRoot(name: string, root: Instance)

Registers a search root for Instance coercion. Multiple roots can be registered and are searched in registration order. Required before using "Instance" as a route param type.

TC.RegisterInstanceRoot("Workspace", workspace)
TC.RegisterInstanceRoot("Players", game:GetService("Players"))

TC.ToString(workspace.Baseplate)           -- "Part:Baseplate"
TC.FromString("Part:Baseplate", "Instance") -- workspace.Baseplate

Supports

TC.Supports(typeName: string) → boolean

Returns true if the type name is recognised. Used by the Router to validate param type annotations at registration time.

TC.Supports("vector3")          -- true
TC.Supports("Enum.CameraType")   -- true
TC.Supports("Enum")             -- true
TC.Supports("banana")           -- false
