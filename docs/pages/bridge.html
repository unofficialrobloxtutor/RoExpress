Bridge

Internal server-side event bus. Decouples modules from each other | a combat module can fire a "playerDied" event without knowing who is listening. Bridge does not cross the client/server boundary.

local Bridge = RoExpress.Bridge

Firing & binding

Bridge:Fire(event, ...)
Bridge:Bind(event, callback) → id
Bridge:Unbind(id)
Bridge:Has(event) → boolean
-- Module A: fires event
Bridge:Fire("playerDied", player, killer)

-- Module B: reacts
Bridge:Bind("playerDied", function(player, killer)
    updateLeaderboard(killer)
end)

Yielding

Bridge:Wait(event) → ...
Bridge:WaitUntil(event, predicate) → ...
Bridge:WaitFirst({events}) → event, ...

Wait yields until the next time the event fires. WaitUntil keeps yielding until the predicate returns truthy. WaitFirst yields until any of the listed events fires and returns which one fired first.

-- Yield until any round-ending event
local event, data = Bridge:WaitFirst({ "roundWon", "roundDraw", "roundTimeout" })
print("Round ended via:", event)
