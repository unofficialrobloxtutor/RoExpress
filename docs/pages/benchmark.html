Client API

Benchmark

Fires real requests and measures wall-clock round-trip time. Works with RoExpress routes on the main channel, named port channels, and vanilla Roblox RemoteFunction and RemoteEvent — so you can compare everything side by side. Client-only.

Benchmark reuses your existing Network. Pass the instance you already have — no second remote is opened.

Quick start

-- Get the benchmark wired to the main channel
local bench = RoExpress("Benchmark")

-- RoExpress main channel
bench:RunAndPrint("player/123", { method = "GET", iterations = 20 })

-- RoExpress named port
local portNet = RoExpress("Network", "combat")
bench:RunOnPort(portNet, "shoot/123", { method = "POST" })

-- Vanilla RemoteFunction (full round-trip)
local rs = game:GetService("ReplicatedStorage")
bench:RunRemoteFunction(rs:WaitForChild("EchoFunc"), { args = { "hello" } })

-- Vanilla RemoteEvent (fire-only timing)
bench:RunRemoteEvent(rs:WaitForChild("MyEvent"), { args = { "hello" } })

Run

bench:Run(route, config?) → Results

Benchmarks a route on the main RoExpress channel. Runs a warmup pass first (discarded), then iterations measured requests.

RunOnPort

bench:RunOnPort(portNetwork, route, config?) → Results

Same as Run but fires against a named port channel. Obtain portNetwork with RoExpress("Network", "portName").

local combatNet = RoExpress("Network", "combat")
bench:RunOnPort(combatNet, "shoot/123", {
    method = "POST",
    data   = { weapon = "AK47" },
    label  = "combat POST shoot",
})

RunRemoteFunction

bench:RunRemoteFunction(remote, config?) → Results

Full round-trip benchmark against a vanilla RemoteFunction. Calls remote:InvokeServer(...) and measures until the response arrives — directly comparable to a RoExpress route benchmark.

local echoFunc = game:GetService("ReplicatedStorage"):WaitForChild("EchoFunc")
bench:RunRemoteFunction(echoFunc, {
    args       = { "payload" },
    iterations = 30,
    label      = "vanilla RF echo",
})

RunRemoteEvent

bench:RunRemoteEvent(remote, config?) → Results

Fire-only timing for a vanilla RemoteEvent. Measures the client-side cost of FireServer — this is not round-trip; RemoteEvent gives no response. Use RunRemoteFunction if you want a fair comparison against RoExpress routes.

Results from RunRemoteEvent show fire overhead only — they will be much faster than any round-trip benchmark and cannot be directly compared. The target field will read "RemoteEvent (fire only)" to make this clear in Compare output.

Writes a formatted summary to the Studio output window. The header line shows the target type and method so you can identify each run at a glance.

RunAndPrint

bench:RunAndPrint(route, config?) → Results

Convenience wrapper — runs and prints in one call. Returns Results so you can store it for Compare.

bench:RunAndPrint("player/123", { method = "GET", iterations = 30 })

Compare

bench:Compare({ results, ... })

Prints multiple Results side-by-side in a columnar table. The target row shows what each column is measuring so mixed comparisons are easy to read.

local rs        = game:GetService("ReplicatedStorage")
local combatNet = RoExpress("Network", "combat")
local echoFunc  = rs:WaitForChild("EchoFunc")

bench:Compare({
    bench:Run("player/123",                 { label = "Main GET" }),
    bench:RunOnPort(combatNet, "shoot/1",  { method = "POST", label = "Port POST" }),
    bench:RunRemoteFunction(echoFunc,       { label = "Vanilla RF" }),
})

Destroy

bench:Destroy()

Drops the internal Network reference. Does not destroy the Network instance — the caller owns it.

Rate limiting. With interval = 0 and high iteration counts you will see 429 errors in results.errorCodes. Call tokenBucket:Reset() on the server before each run to clear per-player state.
