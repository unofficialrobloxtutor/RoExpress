Router

RoExpress routes are URL-like strings supporting typed named params, wildcards, globs, and inline constraints. The Router is embedded in App and Port | you don't use it directly, but understanding its syntax is essential.

Pattern constraints

Wrap a Lua pattern in () after the param name to restrict which values the segment matches. The pattern is anchored | the entire segment must match.

-- digits only, raw string
app:Get("user/:id(%d+)", function(req, res)
    print(req.params.id)  -- "123" — string
end)

-- digits only, coerced to number
app:Get("user/:id(%d+)=number", function(req, res)
    print(req.params.id)  -- 123 — number
end)

-- lowercase letters only
app:Get("cmd/:action(%a+)", function(req, res)
    print(req.params.action)  -- "fire", "reload" | rejects "fire2"
end)

Uses standard Lua patterns. Common classes: %d digit, %a letter, %w alphanumeric, %l lowercase, %u uppercase. A non-matching segment causes the route to be skipped entirely (404), not a 400.

Wildcards and globs

-- * matches exactly one segment, value in req.captures[1]
app:Get("asset/*/preview", function(req, res)
    print(req.captures[1])  -- "sword", "shield" …
end)

-- multiple * each get their own captures index
app:Get("zone/*/item/*", function(req, res)
    print(req.captures[1], req.captures[2])
end)

-- ** consumes all remaining segments as a table
-- matches: files/a  files/a/b  files/a/b/c …
app:Get("files/**", function(req, res)
    -- req.captures[1] = { "a", "b", "c" }
    print(table.concat(req.captures[1], "/"))
end)

Examples

-- Typed named param
app:Get("shop/:itemId=int", function(req, res)
    print(req.params.itemId)  -- always a whole number
end)

-- Pattern + type: digits only, returned as number
app:Get("page/:n(%d+)=int", function(req, res)
    res:Send({ page = req.params.n })
end)

RouteBuilder — typed URL construction

Router.Route(pattern) returns a RouteBuilder<T> — a small object that pairs a pattern with a type-checked URL builder. It solves two problems at once: your route string and your handler registration can never drift apart, and building redirect or navigation URLs is always serialised correctly through TypeCoercer.

RoExpress.Router.Route(pattern: string) → RouteBuilder<T>
FieldTypeDescription
.patternstringThe raw route string — pass to app:Get / app:Post
.build(params)(T) → stringBuilds a stable URL from typed params using TypeCoercer
Luau limitation. Luau cannot extract param names from a string literal at the type level (no template literal types). You annotate T once with :: RoExpress.RouteBuilder<{...}> and the compiler enforces it everywhere .build() is called.

Basic usage

local RoExpress = require(game.ReplicatedStorage.RoExpress)
local App = RoExpress.GetApp()

local GetCoins = RoExpress.Router.Route("player/:UserId=number/coins") :: RoExpress.RouteBuilder<{
    UserId: number,
}>

-- .pattern keeps registration in sync with the builder
App:Get(GetCoins.pattern, function(req, res)
    res:Send({ coins = 100 })
end)

-- .build() is type-checked and always round-trips correctly
GetCoins.build({ UserId = 12345 })  -- "/player/12345/coins"

Redirecting with typed Enum params

The most common use case — passing typed params through a redirect without the URL breaking. Before RouteBuilder, interpolating an EnumItem into a template string would produce Enum.CameraType.Scriptable, which the router couldn't parse back. .build() uses TypeCoercer internally so this always round-trips.

local Url = {
    SendToServer = RoExpress.Router.Route("/server/:Value=number/:Value2=Enum.CameraType") :: RoExpress.RouteBuilder<{
        Value  : number,
        Value2 : Enum.CameraType,
    }>,
    SendToClient = RoExpress.Router.Route("/client/:Value=number/:Value2=Enum.CameraType") :: RoExpress.RouteBuilder<{
        Value  : number,
        Value2 : Enum.CameraType,
    }>,
}

App:Get(Url.SendToServer.pattern, function(req, res)
    res:Redirect(Url.SendToClient.build({
        Value  = req.params.Value,
        Value2 = req.params.Value2,
    }))
end)

App:Get(Url.SendToClient.pattern, function(req, res)
    print("Fired")
end)

Centralised route table

The recommended pattern — define all your routes in one shared file and import them wherever you register handlers or build URLs. The compiler enforces the param shape at every call site.

-- routes.luau (shared module)
local RoExpress = require(game.ReplicatedStorage.RoExpress)
local R = RoExpress.Router

return {
    Shop  = R.Route("/shop/:PlayerId=number/:ItemId=number") :: RoExpress.RouteBuilder<{
        PlayerId : number,
        ItemId   : number,
    }>,
    Equip = R.Route("/equip/:PlayerId=number/:Slot=string") :: RoExpress.RouteBuilder<{
        PlayerId : number,
        Slot     : string,
    }>,
}

-- server.luau
local Url = require(script.routes)

App:Get(Url.Shop.pattern, function(req, res)
    res:Redirect(Url.Equip.build({ PlayerId = req.params.PlayerId, Slot = "primary" }))
end)

Query strings

Any ?key=value suffix on the route is stripped before matching and made available as req.query. The route pattern itself never includes the query string.

-- client
network:Get("shop/items?limit=10&sort=price")

-- server
app:Get("shop/items", function(req, res)
    print(req.query.limit)  -- "10"  (always string)
    print(req.query.sort)   -- "price"
    res:Send(Shop.GetItems(req.query))
end)

Query values are always strings. Coerce manually if you need a number: tonumber(req.query.limit).

Param error handling

When a typed param fails coercion (wrong type or out of range), the default behaviour is to respond 400 and not call the handler. Override this with app:OnParamError(fn) | see App.

Route priority

Routes are sorted automatically by specificity, not registration order. Literal > constrained > typed > plain param > wildcard > glob. Two routes of equal specificity are matched in registration order.
