Bridge → Bind & Fire

Bind & Fire

Register persistent or one-shot listeners on named events, remove them by id, and fire events to all current listeners with any number of arguments.

API

Bridge:Bind(event: string, callback: (...any) → ()) → id
Bridge:BindOnce(event: string, callback: (...any) → ()) → id
Bridge:Unbind(id)
Bridge:Fire(event: string, ...)

Bind and fire

Bind registers a listener that stays active until explicitly removed. Fire calls all listeners currently bound to that event name.

-- Module A: fires event
Bridge:Fire("playerDied", player, killer)

-- Module B: reacts
Bridge:Bind("playerDied", function(player, killer)
    updateLeaderboard(killer)
end)

One-shot listener

BindOnce registers a listener that automatically removes itself after firing once:

Bridge:BindOnce("roundStarted", function(roundData)
    InitRoundUI(roundData)
end)

Removing a listener

Bind and BindOnce both return an id. Pass it to Unbind to remove the listener at any time:

local id = Bridge:Bind("itemPurchased", function(item)
    LogPurchase(item)
end)

-- Later, when this module is shutting down:
Bridge:Unbind(id)

Shop → inventory example

A shop module fires an event; the inventory module reacts without either module needing a direct reference to the other:

-- ShopModule.lua
local function Purchase(player, itemId)
    -- ... validate, charge currency ...
    Bridge:Fire("itemPurchased", player, itemId)
end

-- InventoryModule.lua
Bridge:Bind("itemPurchased", function(player, itemId)
    AddToInventory(player, itemId)
end)
Bridge is a server-only internal bus. It does not cross the client/server boundary. For server-to-client events use Broadcast; for client-side subscriptions use Listener.
