Client API

Listener

Receives both Broadcast unreliable events and App Push reliable events on the client. A single listener:On() call handles both channels automatically.

local listener = RoExpress("Listener")

Subscribing

listener:On(event, callback) → id
listener:Once(event, callback) → id
ParameterTypeDescription
eventstringEvent name | matches what was passed to Emit or Push on the server
callback(data: any) → ()Called every time the event arrives (On) or only the first time (Once)

Both return a subscription id that can be passed to listener:Off(id).

Unsubscribing

listener:Off(id)
local id = listener:On("roundEnd", function(data)
    print("Winner:", data.winner)
end)

-- Remove when no longer needed
listener:Off(id)

Middleware

listener:Use(id, fn)  ·  listener:Unuse(id)

Client-side middleware runs before every event callback. Return false to suppress the event.

listener:Use("log", function(event, data)
    print("[Listener]", event)
end)

Dual-channel transparency

SourceChannelGuaranteed
broadcast:Emit()UnreliableRemoteEvent✗
app:Push()RemoteEvent✓

Both arrive at the same listener:On(event) handler. You don't need separate handlers for reliable vs unreliable events | Listener routes them automatically by source.

Destroy

listener:Destroy()

Removes all subscriptions and disconnects both RemoteEvent listeners.
