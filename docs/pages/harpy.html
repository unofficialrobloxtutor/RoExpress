Server API

Harpy NEW

Outbound HTTP client for server-to-external-API calls. Set a base URL and shared headers once; every subsequent request uses them automatically. Failed requests retry with exponential backoff. Every method returns a consistent response shape, and async variants return a chainable Promise.

Server only | HttpService:RequestAsync is not available on the client. Harpy will error if called from a LocalScript.

Setup

Harpy is a factory | create one client per external API and configure it once:

local RoExpress = require(game.ReplicatedStorage.RoExpress)
local Harpy     = RoExpress.Harpy

local client = Harpy.New({
    base    = "https://api.example.com",
    retries = 3,   -- optional, default 3
})

client:SetHeader("Authorization", "Bearer " .. SECRET)
client:SetHeader("X-Game-Id",     "my-game-123")

You can also pass headers directly in the config table. All fields are optional:

local client = Harpy.New({
    base    = "https://hooks.slack.com/services/...",
    headers = { ["Content-Type"] = "application/json" },
    retries = 1,
})

Callback style

Every method fires in a new thread and calls the callback when done. Omit the callback to fire-and-forget.

-- GET
client:Get("/players/123", function(res)
    if res.ok then
        print(res.data.name, res.data.score)
    else
        warn("fetch failed", res.status)
    end
end)

-- POST | body table is JSON-encoded automatically
client:Post("/events", {
    type   = "round_end",
    winner = "Player1",
    kills  = 12,
})

-- PUT / DELETE
client:Put("/players/123", { score = 9900 })
client:Delete("/sessions/abc")

Promise style

The *Async variants return a Promise. Resolves with the Response on 2xx, rejects with { message, status, response } on failure.

client:GetAsync("/leaderboard")
    :Then(function(res)
        return res.data   -- pass data to the next Then
    end)
    :Then(function(data)
        UpdateLeaderboard(data)
    end)
    :Catch(function(err)
        warn("leaderboard fetch failed", err.message, err.status)
    end)

client:PostAsync("/scores", { userId = userId, score = score })
    :Catch(function(err)
        warn("score submit failed", err.status)
    end)

Response shape

Every callback and resolved Promise receives the same object:

FieldTypeDescription
okbooleanTrue if the status code is 2xx
statusnumberHTTP status code (0 if connection failed)
dataanyJSON-decoded body; raw string if the body is not valid JSON; nil if empty
headers{ [string]: string }Response headers
rawstringRaw response body string

Retry behaviour

On a 5xx response or a connection error, Harpy waits and retries automatically before giving up:

AttemptWait before retry
1 (first try)|
21 s
32 s
4 (default max)4 s

4xx responses (client errors) are not retried | they are returned immediately.

API reference

Constructor

SignatureReturnsDescription
Harpy.New(config?) Harpy Create a new client. config fields: base (string), headers (table), retries (number, default 3).

Configuration methods

MethodDescription
:SetBase(url)Set or replace the base URL. Applied to all future requests.
:SetHeader(key, value)Add or update a persistent header sent on every request.
:RemoveHeader(key)Remove a persistent header.
:SetRetries(n)Change the maximum retry count. Use 0 to disable retries.

Callback methods

MethodDescription
:Get(path, callback?)GET request. Callback receives Response when done. Omit callback to fire-and-forget.
:Post(path, body, callback?)POST request. body table is JSON-encoded automatically. Adds Content-Type: application/json.
:Put(path, body, callback?)PUT request.
:Delete(path, callback?)DELETE request.

Promise methods

MethodDescription
:GetAsync(path)Returns a Promise. Resolves with Response on 2xx.
:PostAsync(path, body)Returns a Promise.
:PutAsync(path, body)Returns a Promise.
:DeleteAsync(path)Returns a Promise.

Real-world examples

Webhook (Discord / Slack)

local webhook = Harpy.New({ base = DISCORD_WEBHOOK_URL })

local function NotifyRoundEnd(winner: string, kills: number)
    webhook:Post("", {
        content = string.format("Round over! Winner: **%s** (%d kills)", winner, kills)
    })
end

External leaderboard API

local api = Harpy.New({ base = "https://scores.example.com" })
api:SetHeader("X-Api-Key", API_KEY)

local function SubmitScore(userId: number, score: number)
    api:PostAsync("/scores", { userId = userId, score = score })
        :Catch(function(err)
            warn("Score submit failed after retries:", err.status)
        end)
end

local function GetTop10(): { any }
    local data
    api:GetAsync("/scores/top?limit=10")
        :Then(function(res) data = res.data end)
        :Finally(function() -- cleanup if needed end)
    return data
end

Multiple APIs in the same game

-- Each client is independent | different base URL, different headers
local analyticsClient = Harpy.New({ base = "https://analytics.example.com" })
local webhookClient   = Harpy.New({ base = DISCORD_WEBHOOK_URL, retries = 1 })
local scoresClient    = Harpy.New({ base = "https://scores.example.com" })

analyticsClient:SetHeader("Authorization", "Bearer " .. ANALYTICS_SECRET)
scoresClient:SetHeader("X-Api-Key", SCORES_KEY)
Each Harpy client is completely independent. Base URL, headers, and retry count are per-instance. Create as many as you need.
