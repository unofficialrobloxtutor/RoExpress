Roadmap

What's being built next for RoExpress — ordered by priority, nearest at the top. Each item links to the relevant section of the docs so you can see what's changing.

Status key  —  ● Next up — in active development    ● Planned — scoped and sequenced    ● Future — on the horizon
Automatic Retry NEXT
Failed requests handle themselves gracefully without developer intervention
  • Exponential backoff on network errors and timeouts — 1s → 2s → 4s
  • Configurable max attempts globally at construction or overridden per request
  • Transparent to the caller — promises and callbacks resolve normally on retry success
  • Affects the Network request/response path
Compact Handler Arguments SHIPPED v2.3
Handler signatures slim from four arguments down to two — no migration required
  • Current: app:Get("/route", function(Player, Payload, req, res))
  • New compact: app:Get("/route", function(req, res))
  • req.player and req.raw available on the request object for cases that need them
  • Uses debug.info parameter count at dispatch — existing four-arg handlers continue working without changes
  • Affects App and Ports
Deflate & LZH Compression PLANNED
Codec upgraded from LZ77 to full Deflate — LZ77 sliding window plus Huffman entropy coding
  • Huffman entropy coding stage added on top of the existing LZ77 pass — meaningfully better ratios on repetitive JSON payloads
  • Hash-chain match lookup replaces the O(n×w) inner loop — faster at large payload sizes
  • Same opt-in API — { compress = true } per route, nothing else changes for existing users
  • Based on Huffman (1952) and Ziv & Lempel (1977) — see Research Papers
  • Affects Codec
Stream Overhaul — Generic Channels PLANNED
Stream rebuilt as typed channels rather than a hardcoded FPS singleton
  • Schema-defined channels — declare field names and types, Stream handles buffer pack/unpack automatically
  • Works for anything: wheel rotation, physics state, weather, economy — not only shooters
  • Batch broadcast model retained — one buffer, one FireAllClients per tick regardless of record count
  • Delta compression built in — only records that changed since the last tick are sent
  • FPS-specific features (hit validation, lag compensation) promoted to a Stream.Combat layer built on top
  • Affects Stream and the FPS Stream guide
Improved Typing PLANNED
Better type exports for a cleaner intellisense experience across the whole surface
  • NetworkResponse, NetworkPayload, and other internal types exported from the public module surface
  • Typed handler callback signatures — no more any in route parameters
  • Affects the Types reference
YouTube Showcase — Gun Framework FUTURE
Full gun framework built live on YouTube demonstrating RoExpress end to end
  • Routes for shooting, reloading, and damage validation; Stream for hit reporting; Codec for match state responses
  • Shows the complete pipeline from client request to server validation in a single real project
  • Scheduled after the Stream overhaul ships so the video reflects the final API
  • YouTube channel
