Server API

App

Handles all incoming client requests. Owns its own Router and TokenBucket instance — no shared global state.

local app = RoExpress.GetApp()   -- typed
-- or: RoExpress("App")          -- returns any

Routing

GET app:Get(route, handler, options?)
POST app:Post(route, handler, options?)
PUT app:Put(route, handler, options?)
DELETE app:Delete(route, handler, options?)
ParameterTypeDescription
routestringSupports typed params, wildcards, globs, inline constraints. See Router.
handlerfunctionModern: (req, res) → ()  ·  Legacy: (Player, Payload, req, res) → (). Detected automatically by parameter count.
options.compressboolean?Enable LZ77 compression on the response. GET/POST only. Default false.

Handler signatures

Two calling conventions are supported. RoExpress detects which one to use automatically via debug.info.

-- Modern (recommended) — req.player and req.raw are populated
app:Get("player/:id=number", function(req, res)
    res:Send({ id = req.params.id, player = req.player.Name })
end)

-- Legacy — Player and Payload passed as first two args
app:Get("player/:id=number", function(Player, Payload, req, res)
    res:Send({ id = req.params.id })
end)

req object

FieldTypeDescription
req.params{[string]: any}Named route params, coerced to declared type
req.captures{any}Positional wildcard / glob captures

res object

MethodDescription
res:Send(data?)Send 200 success response. PUT/DELETE accept boolean? or nil only — passing a table is warned and stripped. Callable once.
res:Error(message)Send 500 error response. Callable once.
res:Status(code)Set HTTP-style status code. Chainable — res:Status(204):Send()

Middleware

app:Use(id, fn)  ·  app:Unuse(id)

Return false to block with 403. Throw to send 500. Return nothing to continue. Middleware runs in registration order before every handler.

app:Use("auth", function(Player, Payload)
    if not isAuth(Player) then return false end
end)

app:Use("logger", function(Player, Payload)
    print(Player.Name, Payload.method, Payload.route)
end)

Server push

PUSH app:Push(player, event, data)
PUSH app:PushAll(event, data)
PUSH app:PushTo(players, event, data)

Reliable server-to-client events over the RemoteEvent. Received by listener:On(event). Guaranteed delivery — does not use the unreliable channel.

Named ports

app:Listen(name, callback, settings?)
app:GetPort(name) → Port

Create an isolated pipeline with its own RemoteEvent, Router, and TokenBucket. See Ports.

Other

app:OnParamError(fn)

Called when a typed route param fails coercion. If not registered, failures send 400 automatically.

app:Destroy()

Disconnects the remote and destroys all registered ports.

app.TokenBucket — direct access to the rate limiter instance. See TokenBucket.
