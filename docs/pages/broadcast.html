Server API

Broadcast

Server-to-client unreliable events. Use Broadcast for high-frequency updates where an occasional dropped frame is acceptable — damage numbers, movement indicators, visual effects. Use App Push for guaranteed delivery.

local broadcast = RoExpress("Broadcast")

API

broadcast:Emit(player, event, data)
broadcast:EmitAll(event, data)
broadcast:EmitTo(players, event, data)
ParameterTypeDescription
playerPlayerTarget player for Emit
players{Player}Target subset for EmitTo
eventstringEvent name — matched by listener:On(event)
dataanyPayload. Serialised over UnreliableRemoteEvent. Must be ≤ 900 bytes.

Receiving on the client

Broadcast events are received by Listener:

-- Server
broadcast:EmitAll("hit", { dmg = 35, pos = hit.Position })

-- Client
local listener = RoExpress("Listener")
listener:On("hit", function(data)
    spawnDamageNumber(data.pos, data.dmg)
end)

Rate limiting

Broadcast shares the global TokenBucket. Large volumes of emits within a frame are automatically throttled. If you need per-event limits, use a Port with its own TokenBucket settings.
