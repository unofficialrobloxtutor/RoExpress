TokenBucket

Per-player rate limiter. Used internally by App and each Port. Also available standalone for custom rate-limiting logic outside the request pipeline.

-- Access App's built-in bucket
local bucket = app.TokenBucket

-- Or create a standalone instance
local TokenBucket = RoExpress("TokenBucket")
local myBucket    = TokenBucket.new({ max = 10, refill = 5 })

Construction

TokenBucket.new(options) → TokenBucket
OptionDefaultDescription
max60Maximum tokens per player
refill20Tokens added per second
startmaxInitial token count when a player first appears
cost1Tokens consumed per Consume() call

API

bucket:Consume(player) → boolean

Attempts to consume cost tokens from the player's bucket. Returns true on success, false if empty (rate limit hit). This is what the request pipeline calls automatically | you don't usually need to call it directly.

bucket:HasTokens(player) → boolean

Check without consuming. Useful for preemptive UI feedback.

bucket:Grant(player, amount)
bucket:GrantAll(amount)
bucket:GrantExact(player, amount)

Grant adds tokens (capped at max). GrantAll adds to every active player. GrantExact sets the count directly.

bucket:Reset(player)

Resets a player's bucket to the start value.

bucket:Destroy()

Stops the refill loop and frees all state.

Example: custom action cost

-- Port with high-cost actions consuming 5 tokens each
app:Listen("economy", function(port)
    port:Post("trade", handler)
end, { max = 50, refill = 5, cost = 5 })

-- Player can burst 10 trades then must wait 1 second per trade
