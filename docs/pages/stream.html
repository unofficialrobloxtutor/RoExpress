Shared Module

Stream

Schema-defined typed binary channels multiplexed over two shared remotes. No JSON, no Base64. Both server and client define identical channels — IDs are hash-derived, so there is no ordering dependency between registration calls.

Breaking change from v2.1. The old FPS-specific singleton API (SendState, OnHit, EnableLagCompensation, etc.) has been removed. Stream is now a general-purpose binary channel system.

Wire format

All channels share two remotes in ReplicatedStorage/RoExpressStream/:

RemoteTypeWire format
StreamUnreliableUnreliableRemoteEvent[channelId:u16][seq:u16][payload]
StreamReliableRemoteEvent[channelId:u16][payload]

Bit 15 of channelId is the delta flag. Bits 0–14 are the channel ID derived from a djb2 hash of the channel name.

Quick start

-- Both server and client define the same channels
local Stream = RoExpress.Stream

local move = Stream.Channel("playerMove", Stream.Schema({
    { "pos",   "Vector3" },
    { "vel",   "Vector3" },
    { "state", { "flags", "jumping", "sprinting" } },
}))

Stream.Init()  -- call once after all Channel() definitions

-- Server: subscribe
move:On(function(data, player)
    print(player.Name, data.pos, data.state.jumping)
end)

-- Client: send
move:Send({
    pos   = hrp.Position,
    vel   = hrp.AssemblyLinearVelocity,
    state = { jumping = false, sprinting = true },
})
Order matters. Stream.Channel() must be called before Stream.Init(). Define all channels first, then init once on both server and client.

Stream.Channel()

Stream.Channel(name: string, schema: Schema, options?: Options) → Channel

Defines a typed binary channel. Returns a Channel object used to send and subscribe.

OptionTypeDefaultDescription
reliableboolean?falseUse RemoteEvent instead of UnreliableRemoteEvent
maxRatenumber?—Max incoming fires/sec per player (server-side)
onDropfn?—Called when a packet is rate-limited
deltaIntervalnumber?10Force a full resync every N delta packets

Stream.Schema()

Stream.Schema(definition: {{ string | {any} }}) → Schema

Compiles a schema from an ordered array of field definitions. Fixed-size schemas (no string fields) pre-compute byte offsets and are eligible for delta compression.

-- Fixed-size — all fields have known byte widths
local posSchema = Stream.Schema({
    { "pos",     "Vector3" },   -- 12 bytes
    { "health",  "u16"     },   --  2 bytes
    { "alive",   "bool"    },   --  1 byte
})

-- Variable-size — string field makes it non-fixed
local chatSchema = Stream.Schema({
    { "name",    "string"  },
    { "message", "string"  },
})

Server send API

MethodDescription
channel:SendTo(player, data)Send to one player
channel:SendExcept(except, data)Send to all players except one
channel:Broadcast(data)FireAllClients — one remote call
channel:SendToList(players, data)Send to a specific list
channel:SendToDelta(player, data)Delta-compressed to one player
channel:BroadcastDelta(data)Delta-compressed to all players

Client send API

channel:Send(data: table)

Fires to the server. Client-only — will assert if called on the server.

Subscribe API

Works identically on both server and client.

-- Persistent subscription — fires every time
local unsub = channel:On(function(data, sender)
    -- sender is a Player on the server, nil on the client
    print(data.pos)
end)

-- One-shot — auto-unsubscribes after first fire
local unsub = channel:Once(function(data, sender)
    print("first packet from", sender)
end)

unsub()  -- unsubscribe at any time

Delta compression

Only changed fields are sent. Requires a fixed-size schema (no string fields). Fields are compared at compile-time byte offsets — no per-field overhead at runtime.

-- Per-player delta (each player has its own last-known state)
channel:SendToDelta(player, newData)

-- All-players delta (each player may differ — joined at different times)
channel:BroadcastDelta(newData)

A full packet is forced on the first send and every deltaInterval packets (default 10). This ensures clients resync even after unreliable packet loss.

Custom types

Register additional types before calling Stream.Init().

Stream.Types.Register("hp", {
    size  = 2,
    write = function(buf, offset, value)
        buffer.writeu16(buf, offset, value)
    end,
    read  = function(buf, offset)
        return buffer.readu16(buf, offset)
    end,
})

Rate limiting

Per-channel rate limiting applies on the server for client→server packets only.

local move = Stream.Channel("playerMove", schema, {
    maxRate = 30,  -- max 30 fires/sec per player
    onDrop  = function(player, reason)
        warn(player.Name, "dropped:", reason)
    end,
})

Full example — player movement

-- shared.luau (required on both server and client)
local RoExpress = require(game.ReplicatedStorage.RoExpress)
local Stream    = RoExpress.Stream

local moveSchema = Stream.Schema({
    { "pos",    "Vector3"                       },
    { "vel",    "Vector3"                       },
    { "cf",     "CFrameLight"                   },
    { "state",  { "flags", "jumping", "sprinting", "crouching" } },
    { "health", "u16"                           },
})

local move = Stream.Channel("playerMove", moveSchema, {
    maxRate       = 30,
    deltaInterval = 15,
})

Stream.Init()

return { move = move }

-- server.luau
local channels = require(script.Parent.shared)

channels.move:On(function(data, player)
    -- rebroadcast to all other clients (delta-compressed)
    channels.move:SendToDelta(player, data)
end)

-- client.luau
local channels = require(script.Parent.shared)
local hrp      = LocalPlayer.Character.HumanoidRootPart

RunService.Heartbeat:Connect(function()
    channels.move:Send({
        pos    = hrp.Position,
        vel    = hrp.AssemblyLinearVelocity,
        cf     = hrp.CFrame,
        state  = { jumping = isJumping, sprinting = isSprinting, crouching = false },
        health = humanoid.Health,
    })
end)

-- receive world-state updates from server
channels.move:On(function(data, _)
    -- apply to other players' characters
end)
