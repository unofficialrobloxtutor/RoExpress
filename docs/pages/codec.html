Codec

LZ77 and LZH (Deflate) compression for large payloads. Used internally by App when options.compress = true and auto-decompressed on the client. Also available for manual use.

local Codec = RoExpress.Codec

Algorithms

AlgorithmFunctionNotes
LZ77Codec.LZ77Sliding window. Fast, good ratio on repetitive data.
LZH / DeflateCodec.LZHLZ77 + Huffman coding. Better ratio, slightly more CPU. Added in v2.3.

API

Codec.LZ77:Compress(data: string) → string
Codec.LZ77:Decompress(data: string) → string
Codec.LZH:Compress(data: string) → string
Codec.LZH:Decompress(data: string) → string
Codec:IsCompressed(data: string) → boolean

Manual usage

local Codec  = RoExpress.Codec
local json   = game:GetService("HttpService"):JSONEncode(bigTable)
local packed = Codec.LZH:Compress(json)

print(#json, "→", #packed)  -- original vs compressed size

local back = Codec.LZH:Decompress(packed)
local table = game:GetService("HttpService"):JSONDecode(back)

Automatic compression via App

Pass compress = true in route options. The client receives the response decompressed automatically — NetworkResponse.compressed is set to true so you know it was applied:

-- Server
app:Get("map/data", function(req, res)
    res:Send(largeMapTable)
end, { compress = true })

-- Client — data is already decompressed
network:Get("map/data", nil, function(res)
    print(res.compressed)  -- true
    loadMap(res.data)
end)

When to use which

ScenarioRecommendation
Large repeated-structure data (map tiles, inventories)LZH — better ratio
High frequency small updatesDon't compress — overhead not worth it
Binary buffer dataUse Stream instead
Need to check before decompressCodec:IsCompressed(data)
