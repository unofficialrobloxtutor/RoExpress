Server API

Tamper

Passive exploit detection layer. Tamper observes the request pipeline and accumulates strikes against players who show suspicious behaviour. It never blocks requests itself | blocking is done by the rate limiter and middleware. Tamper is a singleton.

local Tamper = RoExpress("Tamper")

What it detects

SignalDescription
Version spoofingClient claims a version not matching the server expectation
Malformed payloadMissing required fields, wrong types, or invalid method strings
Route scanningRepeated requests to non-existent routes in a short window
Rate floodingToken bucket exhaustion attempts | firing far beyond the allowed rate

Strike system

Tamper:GetStrikes(player) → number
Tamper:AddStrike(player)
Tamper:ClearStrikes(player)
Tamper:OnStrike(callback)

Subscribe to OnStrike to implement custom logic | log to DataStore, kick immediately, or apply a temp ban. If you don't set a custom handler, the default behaviour kicks the player at 3 strikes.

Tamper:OnStrike(function(player, strikes, reason)
    warn("Strike", strikes, "for", player.Name, "| reason:", reason)
    if strikes >= 3 then
        player:Kick("Exploit detected.")
    end
end)

Configuration

Tamper:Configure(options)
OptionDefaultDescription
maxStrikes3Strikes before the default handler kicks the player
scanWindow10Seconds for route-scan detection window
scanThreshold5Unique 404s in scanWindow to count as scanning

Important note

Tamper is a detection tool, not a prevention tool. Every request still passes through the full rate-limiting and middleware pipeline regardless of strike count. Use OnStrike to take enforcement action | Tamper will not do it automatically unless you rely on the default 3-strike kick.
